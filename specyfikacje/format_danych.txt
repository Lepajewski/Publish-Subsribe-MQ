kolejność danych w przesyłanych danych:
1. nr sygnału (enum sygnałów z brokera)
    sygnały:

    - connack od klienta
        broker czekuje czy już istnieje połączenie od danego ip:port
    - connack od brokera
        2. wygenerowane id klienta                                  (int)

    - ping od brokera i klienta
        pusta wiadomość budząca

    - suback od klienta
        2. długość nazwy topicu                                     (size_t)
        3. nazwa topicu
    - suback od brokera
        2. długość nazwy topicu                                     (size_t)
        3. nazwa topicu
        4. kod powodzenia subskrybcji (jakieś 0 lub 1 czy coś)      (int)
        dla każdej wiadomości w topicu {
            5. id wiadomości                                        (int)
            6. długość wiadomości                                   (size_t)
            7. wiadomość
        }
        8. znak (char)03                                            (char)

    - puback od brokera
        2. id wiadomości                                            (int)
    - puback od klienta
        2. długość nazwy topicu                                     (size_t)
        4. nazwa topicu
        5. długość wiadomości                                       (size_t)
        7. wiadomość
    
    - newmes od brokera
        2. id wiadomości                                            (int)
        4. długość nazwy topicu                                     (size_t)
        6. nazwa topicu
        7. długość wiadomości                                       (size_t)
        9. wiadomość

    - unsuback od klienta
        2. długość nazwy topicu                                     (size_t)
        4. nazwa topicu
    - unsuback od brokera
        2. kod powodzenia desubskrybcji (jakieś 0 lub 1 czy coś)    (int)

    - disconnack od klienta
        pusty sygnał
    - disconnack od brokera
        pusty sygnał
