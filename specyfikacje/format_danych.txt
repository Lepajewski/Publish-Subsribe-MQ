kolejność danych w przesyłanych danych:
1. nr sygnału (enum sygnałów z brokera)
    sygnały:

    - connack od klienta                                            D - D
        pusta wiadomość
    - connack od brokera                                            D - D
        2. wygenerowane id klienta                                  (int)

    - ping od brokera i klienta                                     ND - ND
        pusta wiadomość budząca

    - suback od klienta                                             D - D
        2. długość nazwy topicu                                     (size_t)
        3. nazwa topicu
    - suback od brokera                                             D - ND
        2. kod powodzenia subskrybcji (jakieś 0 lub 1 czy coś)      (suback_success_code)
        ----- tylko jeśli 2. nie jest SUBACK_FAILURE
        3. długość nazwy topicu                                     (size_t)
        4. nazwa topicu
        5. ilość wiadomości w topicu                                (size_t)
        dla każdej wiadomości w topicu {
            6. id wiadomości                                        (int)
            7. długość wiadomości                                   (size_t)
            8. wiadomość
        }

    - puback od brokera                                             D - ND
        2. id wiadomości                                            (int)
    - puback od klienta                                             D - D
        2. długość nazwy topicu                                     (size_t)
        4. nazwa topicu
        5. długość wiadomości                                       (size_t)
        7. wiadomość
    
    - newmes od brokera                                             D - ND
        2. długość nazwy topicu                                     (size_t)
        3. nazwa topicu
        4. id wiadomości                                            (int)
        5. długość wiadomości                                       (size_t)
        6. wiadomość

    - unsuback od klienta                                           D - D
        2. długość nazwy topicu                                     (size_t)
        4. nazwa topicu
    - unsuback od brokera                                           D - ND
        2. kod powodzenia desubskrybcji                             (unsuback_success_code)

    - disconnack od klienta                                         D - D
        pusty sygnał
    - disconnack od brokera                                         D - ND
        pusty sygnał
